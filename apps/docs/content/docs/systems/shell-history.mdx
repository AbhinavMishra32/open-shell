Web history is route-centric. Product shells often move among files, conversations, settings, inspectors,
and slot tabs without every transition becoming a URL. `useShellHistory` provides one typed stack for that
movement.

<ComponentPreview name="shell-history" />

## Contract

Every entry has `id`, `type`, optional `title`, and optional payload. Duplicate current entries are ignored,
forward history is truncated after a new push, and a maximum stack length can be configured.

Wrap deeply nested controls in `ShellHistoryProvider`, then read the controller with
`useShellHistoryContext`. Router synchronization remains application-owned.
