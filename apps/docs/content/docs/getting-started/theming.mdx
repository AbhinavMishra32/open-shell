---
title: Theming
description: Use Opaline tokens in light, dark, translucent, and embedded surfaces.
---

Opaline styles are built from CSS custom properties. Override tokens at an application root rather than
forking component CSS.

```css
.my-app {
  --opaline-background: #171816;
  --opaline-foreground: #f1f2ee;
  --opaline-border: rgba(255, 255, 255, 0.12);
  --opaline-accent: #7eaee8;
}
```

Use `color-scheme` and your own theme class to coordinate native controls and Opaline surfaces:

```css
.my-app[data-theme="dark"] {
  color-scheme: dark;
}
```

The docs themselves are an example: Fumadocs layout tokens are mapped onto a restrained Opaline palette,
while every live preview uses the package stylesheet directly.
