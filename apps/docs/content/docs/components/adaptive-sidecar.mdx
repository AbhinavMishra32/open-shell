`AdaptiveSidecarLayout` is the shared system for knowledge cards, research summaries, inspectors, and other contextual surfaces that should stay close to the primary workspace without permanently shrinking it.

<ComponentPreview name="adaptive-sidecar" />

## Layout modes

The layout measures its own container with `ResizeObserver` and chooses one of three modes:

- **Overlay** keeps the primary surface fixed and places the sidecar above it.
- **Shift** moves the primary surface by half the sidecar width so both remain visually balanced.
- **Gutter** reserves a real 300px column when the parent has enough room.

The default thresholds and motion mirror Opaline's researched pinned-summary behavior: a 300px surface, 16px separation, and a restrained 300ms spring with almost no bounce.

## Persistent content

The sidecar stays mounted by default. Closing it animates to `translateX(100%)`, `scale(.8)`, and zero opacity while disabling pointer events. This preserves internal state, scroll position, editors, and active tools across repeated toggles.

Set `keepMounted={false}` when a disposable popover is more appropriate.

## Surface composition

`AdaptiveSidecarSurface` provides the visual shell and optional collapse, pin, close, and footer controls. Its content remains application-owned, so the same primitive can host knowledge, source lists, process status, environment controls, or review output.
